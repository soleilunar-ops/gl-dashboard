# 물류 메뉴 — 기능 설명 & PM 전달 요약

> 작성 기준: 리포 코드 상태(2026-04-19~20) · 사이드바 `물류` 4개 메뉴 · **밀크런 Prisma 제거·`/api/allocations`는 Supabase 서버 클라이언트** 반영

## 1. 사용자용 — 핵심 기능 쉬운 설명

### 1.1 총재고 현황 (`/logistics`)

**한 줄 요약:** 지금 창고·쿠팡 쪽 재고와 오늘 입출고 흐름을 한 화면에서 본다.

| 구분          | 설명                                                                                                                                                     |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **통합 현황** | GL 품목 재고 요약과, 오늘 `orders` 기준 외부 입·출고 수량, 쿠팡 센터 재고 스냅샷(`inventory_operation` 최신일) 요약 카드를 같이 본다.                    |
| **GL창고**    | 품목별 현재 수량·금액 표. 기간별 **일말 재고 추이 차트**(`orders` 입·출고 막대 + 현재 총재고 역산 선). 행을 누르면 해당 품목 **입출고 원장** 다이얼로그. |
| **쿠팡센터**  | 판매자센터 재고 CSV 업로드·센터별 목록·발주 상태 열·검색; 행/분석 진입 시 재고 차트·SKU 분석(LLM 서술 등, 별도 API·다이얼로그).                          |

**데이터 출처(개념):** 활성 품목의 현재고는 Supabase 뷰 `v_current_stock`을 중심으로 하고, ERP·쿠팡 코드 표시는 `item_erp_mapping`, `item_coupang_mapping`으로 붙인다. (구 엑셀 스냅샷 병합 방식은 제거된 이력이 있음.)

---

### 1.2 수입 리드타임 (`/logistics/leadtime`)

**한 줄 요약:** 중국·수입 발주 한 건씩 **단계(선적·통관·입고 등)** 일정과 BL을 수기로 쌓고, BL이 있으면 서버 API로 통관 쪽 정보를 보조 조회한다.

| 구분          | 설명                                                                                                                                   |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **추가·수정** | 발주번호, 품목, 단계별 실제/예상일, 승인 여부 등을 **Supabase `import_leadtime` 테이블**에만 저장한다. (과거 MOCK 전용 모드는 제거됨.) |
| **BL**        | BL 번호 입력 후 조회 버튼 → Next.js `/api/tracking` 등 서버 경로(유니패스·공공데이터 키는 서버 환경변수).                              |
| **엑셀**      | 화면에 보이는 목록을 그대로 **xlsx**로 내려받는다(감사·공유용).                                                                        |

---

### 1.3 쿠팡 밀크런 관리 (`/logistics/coupang-milkrun`)

**한 줄 요약:** 티켓팅일(출고일)에 **어느 FC에 파렛트를 몇 개** 넣을지 시뮬레이션하고, 확정한 배정을 DB에 남긴 뒤 기간별로 다시 본다.

| 탭              | 설명                                                                                                                                                                                                                                                      |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **비용 계산기** | 센터 목록·BASIC 단가는 `src/data/milkrun-centers.json`(루트 엑셀에서 `npm run parse:milkrun-centers`로 갱신 가능). 검색·즐겨찾기·필터 단위 일괄 선택, 센터별 파렛트 수 입력 → 줄별 비용·비중·합계·CSV. **작성 중 초안**은 브라우저 `localStorage`에 저장. |
| **기간별 조회** | 출고일 구간으로 저장된 배정을 목록·일별 막대 차트로 보고, 상세·삭제·**센터 단위까지 풀린 CSV**보내기.                                                                                                                                                     |

**저장·조회 방식:** 화면은 **`useMilkrunAllocations`** 가 브라우저 Supabase 클라이언트(로그인 세션·RLS)로 `allocations` / `allocation_items`에 insert·select·delete. 동일 스키마를 쓰는 **선택 REST** `POST|GET /api/allocations`, `GET|DELETE /api/allocations/[id]` 는 **`@/lib/supabase/server`의 `createClient()`** 로 접근하며 **쿠키 세션 인증(401)** 후 동일 테이블을 다룬다. **Prisma·`DATABASE_URL`·`src/lib/db.ts`는 제거됨** (팀 pre-commit 금지 패키지와 정합).

**참고:** 예전에는 이 화면 안에 “재작업일 날씨” 탭이 있었으나, **별도 메뉴**로 분리됨(아래 1.4).

---

### 1.4 재작업일 날씨 (`/logistics/milkrun-weather`)

**한 줄 요약:** **출고일(스케줄 기준일)** 을 넣으면, 그에 맞춰 잡는 **재작업 D-2 / D-1** 날짜의 파주 기상청 예보를 보여 주고, 같은 페이지에서 **파주 앞으로 약 11일 일별 예보**와 **중국 연휴(내장 데이터)** 를 같이 본다.

| 구분               | 설명                                                                                                                                                                                                                         |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **재작업일 카드**  | 파주 격자 고정. 재작업일이 오늘부터 3일 이내면 **단기예보**, 그 밖이면 **중기**(육상·기온 등)로 분기. API는 서버 `KMA_SERVICE_KEY`.                                                                                          |
| **파주 일별 예보** | 단기·중기 호출을 묶어 오늘~D+10 구간을 채운다(`GET /api/weather/forecast` 등).                                                                                                                                               |
| **중국 연휴**      | 국무원 일정을 **`cn-official-holidays-builtin`** 등 코드 내장 데이터로 표시(연휴 구간·D-day·브릿지). 외부 공휴일 API는 정확도 이슈로 제거된 이력 있음. 연도 갱신은 소스 편집; 향후 설정 UI는 미구현(필요 시 PM·기획과 별도). |

## 2. PM 전달 — 추가·정리한 DB·백엔드

아래는 **원격 Supabase에 마이그레이션이 실제 적용됐을 때** 의미가 있다. 로컬에만 파일이 있고 `db push`가 안 됐다면 테이블이 없어서 화면에서 오류 안내가 날 수 있다.

### 2.1 수입 리드타임

| 대상           | 파일(타임스탬프 순)                                      | 내용                                                                     |
| -------------- | -------------------------------------------------------- | ------------------------------------------------------------------------ |
| 테이블 생성    | `20260418123000_create_import_leadtime_table.sql`        | `public.import_leadtime` + 인덱스 + RLS(`authenticated` 전체 허용 패턴). |
| 컬럼 보강      | `20260418104500_add_import_leadtime_manual_expected.sql` | 테이블이 이미 있을 때만 `step1_expected`, `step3_expected` 추가.         |
| PostgREST 권한 | `20260420120000_import_leadtime_api_grants.sql`          | `authenticated`에 DML, `service_role`에 ALL `GRANT`(테이블 없으면 스킵). |

**PM 체크:** 스테이징/운영 각각에서 위 3개가 반영됐는지, 앱이 **로그인 후** Supabase 클라이언트로만 쓰는지 QA.

---

### 2.2 쿠팡 밀크런 배정 저장

| 대상   | 파일                                            | 내용                                                                                                                                                                    |
| ------ | ----------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 테이블 | `20260418150000_create_milkrun_allocations.sql` | `public.allocations`(출고일·총비용·총파렛트·센터 수·메모·타임스탬프), `public.allocation_items`(센터별 단가·파렛트·라인비용), FK + CASCADE 삭제, 인덱스, RLS 동일 패턴. |

**PM 체크:** 비용 계산기 “저장”과 기간별 조회가 이 테이블을 쓰므로, **미적용 시** 저장·목록이 실패한다.

**코드 정리(2026-04-19 이후):** 밀크런 DDL은 **`supabase/migrations/20260418150000_create_milkrun_allocations.sql`만** 유지. **`prisma/` 디렉터리·`@prisma/client`·`src/lib/db.ts`는 삭제**되었고, `/api/allocations`는 위와 동일 Postgres 테이블에 **Supabase JS(서버 세션)** 로만 접근한다.

### 2.5 환경 변수(물류 관련 요약)

| 변수                              | 용도                                                                                                                                                          |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `NEXT_PUBLIC_SUPABASE_*`          | 브라우저·서버 라우트 Supabase 클라이언트(총재고·리드타임·밀크런·`/api/allocations` 등). 일부 API는 추가로 `SUPABASE_SERVICE_ROLE_KEY` 사용(문서·코멘트 참고). |
| `KMA_SERVICE_KEY`                 | 기상청 단기·중기(재작업일·파주 예보).                                                                                                                         |
| `UNIPASS_API_KEY` / 공공데이터 키 | BL 추적 `/api/tracking` (서버).                                                                                                                               |
| `DATABASE_URL`                    | **밀크런과 무관.** (선택) 다른 Postgres 도구용. `.env.example`에 빈 값·주석만 유지.                                                                           |

### 2.6 문서와의 차이(정리)

- `docs/coupang-milkrun-features.md` — **Prisma 제거·2탭 UI·`/api/allocations` = Supabase 서버 세션** 기준으로 갱신됨.
- `docs/logs/pm/import_leadtime.md` — **MOCK** 문구가 남아 있을 수 있으나 코드는 **`NEXT_PUBLIC_LEADTIME_MOCK` 미사용**.
- 과거 로그(`docs/logs/진희.md` 등)에 **밀크런 = `DATABASE_URL` + Prisma migrate** 안내가 남아 있을 수 있음 → **현재는 `supabase/migrations` + `NEXT_PUBLIC_SUPABASE_*`** 가 정본.
