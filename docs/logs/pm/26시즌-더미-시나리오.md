# 26시즌 더미 데이터 시나리오 (개발·LLM 테스트용)

> ⚠️ **이 데이터는 합성 더미입니다.** LLM 시즌 브리프·튜닝 제안 워크플로 테스트를 위해
> 25시즌 실측을 365일 shift + 소폭 변동으로 생성. 실제 쿠팡 판매·기상 데이터 아님.
> 작성: 2026-04-21 (PM 지호)

## 목적

LLM 기능(`generate-season-brief`, `propose-trigger-tuning`) 구현 검증을 위해
25·26 두 시즌 데이터가 필요함. 26시즌은 아직 시작 전(2026-09-01~)이라
실측 없음 → 합성 더미로 채움.

## 생성 규칙

### weather_unified (212 row)

- 25시즌 weather (station='서울', source='asos')를 365일 shift
- 월별 오프셋:
  - 9–10월: +0.3℃ (약간 따뜻)
  - **11–12월: +0.5℃** (온난 — first_freeze 지연·피크 완화)
  - **1월: −1.5℃** (한파 피크 이동)
  - 2–3월: +0.3℃
- 랜덤 노이즈: ±0.8℃ (uniform)
- 기타 필드(precipitation, humidity 등): 그대로 복제

### daily_performance (7,244 row, 34 SKU × 212일)

- 25시즌 판매 row를 365일 shift
- units/gmv/order/customer/page_views: `원값 × 1.05 × (0.92 ~ 1.08 random)`
- 평균 +5% 성장 의도, 실측 결과 +2.3% (random 분포 영향)

## 제한·주의

1. **키워드 데이터 없음** — 26시즌 기간 `keyword_trends` 미생성 → LLM 브리프에서 "검색 급등 데이터 없음"으로 표시됨. 필요 시 25시즌 shift로 추가 가능.
2. **랜덤 시드 고정 안 함** — 재실행하면 수치 미세 변동. 현재 값은 2026-04-21 생성분.
3. **실측 전환 시 삭제 또는 분리** — 26시즌 시작(2026-09-01)이 오면 실측 데이터가 쌓일 것. 이때 더미는:
   - 옵션 A: 실측이 쌓일 때 `ON CONFLICT UPDATE` 로 자연스럽게 덮어씀
   - 옵션 B: 그 전에 `DELETE WHERE sale_date BETWEEN '2026-09-01' AND '2027-03-31' AND created_at <= '2026-04-21'` 로 정리
