# 지호 — 날씨·쿠팡 판매·키워드 삼각분석

> PM 지호. `/analytics/weatherkey` 영역의 기획·데이터·UI를 PM 역할과 병행하여 주도한다.
> 파일 경계: `src/components/analytics/weatherkey/`, `src/app/(dashboard)/analytics/weatherkey/`

## 1. 목표 (What & Why)

- **한 줄 목표**: 날씨 × 쿠팡 판매량 × 검색 키워드, 세 축의 **동시 상관**을 한 화면에서 보여준다.
- **왜**:
  - 수요 예측(정민)과 프로모션 인사이트(나경)가 각각 존재하지만 **외부 신호(날씨·검색 트렌드)를 판매와 동시에 엮어보는 뷰는 없다.**
  - "비 오면 뭐가 더 팔리나 / 한파 키워드 급등과 판매가 같이 움직였나" 같은 질문에 바로 답할 수 있는 **탐색형 대시보드**가 필요.
- **성공 기준 (초안)**:
  - 기간 선택 + SKU/카테고리 필터로 3축 시계열을 겹쳐볼 수 있다.
  - 상관계수(또는 lag 상관) 수치가 카드로 노출된다.
  - 데이터는 Supabase 뷰/함수로 조회 (프론트 mock 금지).

## 2. 데이터 3축

| 축        | 소스          | 변수/함수                                                                            | 담당/참고                  |
| --------- | ------------- | ------------------------------------------------------------------------------------ | -------------------------- |
| 날씨      | 기상청 ASOS   | `KMA_SERVICE_KEY`, `scripts/kma_asos_backfill.py`, `scripts/kma_backfill_verify.sql` | 진희/정민 기존 자산 재사용 |
| 쿠팡 판매 | 쿠팡 WING API | `COUPANG_ACCESS_KEY`, `COUPANG_SECRET_KEY`, `COUPANG_VENDOR_ID`                      | 나경 파이프라인            |
| 키워드    | 네이버 LAPS   | `NAVER_LAPS_CLIENT_ID`, `NAVER_LAPS_CLIENT_SECRET`                                   | **지호 신규**              |

> 키는 `.env.local` 에만 존재. 커밋 금지. `.env.example` 은 빈 값 + 주석만.

## 3. 아키텍처 계획

```
네이버 LAPS  ─┐
기상청 ASOS  ─┼─▶  Supabase (raw)
쿠팡 WING   ─┘            │
                          ▼
              뷰/RPC (일자 × SKU × 지역)
                          │
                          ▼
          WeatherkeyDashboard  ←  _hooks/useWeatherkeyData
```

- **프론트**: `src/components/analytics/weatherkey/` 내부에만 컴포넌트·훅 작성 (팀 경계 규칙).
- **데이터 훅**: `_hooks/useWeatherkeyData.ts` 스켈레톤으로 시작 (CLAUDE.md 규칙 #8).
- **키워드 수집기**: `scripts/naver_laps_ingest.py` (파이썬, 기존 KMA 스크립트 패턴 참고).
- **상관 계산**: 초안은 프론트에서, 규모 커지면 Supabase RPC 로 이관.
