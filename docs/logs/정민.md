**`/forecast/latest`** (forecast.py:130~143):

**실측 (2026-04-19, Supabase MCP execute_sql):**

```sql
SELECT * FROM public.forecasts LIMIT 1;
-- ERROR: 42P01: relation "public.forecasts" does not exist
```

**동작 시뮬레이션:**

1. `_get_supabase_client()` → OK (환경변수 설정 시)
2. `supabase.table("forecasts")` → 클라이언트 객체 생성
3. `.select("*").order().limit()` → 쿼리 빌드
4. `.execute()` → **Postgres 42P01 에러 발생**
5. FastAPI → **500 Internal Server Error**

**결론:** 두 엔드포인트 모두 **100% 죽은 상태**. 호출 시 항상 500. 다만 프론트 `useForecast.ts`는 이 둘을 호출하지 않음(`/daily-sales`, `/weekly-prediction`만 호출) → 실 UI 영향 0.

**FastAPI 기동 상태:** 포트 8000 LISTENING 없음. HTTP 레벨 500 반환 검증 불가. 대신 DB 레벨로 확정.
