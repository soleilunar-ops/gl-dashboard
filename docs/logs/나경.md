### 1. 기능 개요 — 무엇을 만들었나

**프로모션 분석 모듈** (`/analytics/promotion`) — 4탭 BI 대시보드 + 엑셀 업로드 기능.

| 탭             | 역할                                                                     |
| -------------- | ------------------------------------------------------------------------ |
| Effect         | 월별 비용(쿠폰·광고·밀크런·프리미엄) × 매출(GMV) 상관 + 기온 편차 히트맵 |
| ROI            | 채널별 ROI + 마진율 슬라이더 실시간 계산                                 |
| Season Alert   | 누적 비용률 추적 + 1·2차 경고선 + 예측 종료값                            |
| Budget Planner | 목표매출 입력 → 기준시즌 채널비중으로 역산 예산                          |

**공통 입력 경로**: `UploadPanel` → 6종 입력 (CSV/XLSX 4종 + 수동 폼 2종) → DB 8 테이블 → `usePromotion` 훅으로 집계 → 각 탭 소비.

### 5. 데이터 단위 정책 (해결됨 — 중복 아닌 의도된 분리)

#### 배경

초기 리뷰에서 `coupang_daily_performance` ↔ `daily_performance` 가 같은 도메인 데이터를 다른 값으로 저장하는 **중복**으로 판단했음. 엑셀 원본 확인 후 **오판**임을 확인, 정정.

#### 정정된 이해

| 테이블                                     | 출처                                       | 단위                                         | 레벨       |
| ------------------------------------------ | ------------------------------------------ | -------------------------------------------- | ---------- |
| `daily_performance`                        | 쿠팡 Supplier Hub 일별실적 raw             | **박스 단위**                                | 일별 × SKU |
| `coupang_daily_performance` (AGGREGATE 행) | 월별*판매납품*광고비\_현황\_v2.xlsx 시트 ① | **매(낱개) 단위** — 팩수량 × 팩당매수로 환산 | 월별 집계  |

#### 왜 gmv 14~26배 차이가 났나

- 수량: 575,650**매** (낱개) vs 567,392**박스** — 단위 자체가 다름 (우연히 숫자 비슷)
- 금액: 낱개 단가 287원/매 × 수량 = 낱개 기준 판매액 / 박스 단가 × 박스 수 = Supplier Hub GMV → 배수 차이는 **"매 ↔ 박스" 변환 계수** (제품마다 팩당 매수 다르므로 월별 배수도 다름)

#### 분석 목적상 왜 낱개 단위가 필요

24-25시즌과 25-26시즌에 **제품 구성(팩 크기)이 달라짐**. 박스 기준이면 시즌 간 비교 불가. 낱개(매)로 통일해야 "이번 시즌이 작년보다 20% 더 팔림" 같은 비교가 유효.

#### 결론 — 통합 금지, 별도 유지

| 조치    | 내용                                                                                                                                        | 우선순위     |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ------------ |
| 🔴 즉시 | `coupang_daily_performance`, `coupang_delivery_detail` 테이블·컬럼 COMMENT 추가: "수량 단위=매(낱개 환산), 출처=월별\_판매납품 엑셀 Layer1" | 머지 전/직후 |
| 🟡 중기 | 컬럼명 리네이밍 검토: `gmv → sales_amount_ea`, `units_sold → units_sold_ea` (단위 명시)                                                     | 별도 PR      |
| 🟢 유지 | `daily_performance`와 공존. 목적·단위·레벨 모두 다른 별개 자료                                                                              | —            |

#### 초기 판정 vs 수정된 판정 요약

- ❌ Cursor/Claude 초기 분석: "명백한 중복, 머지 전 통합 필수"
- ✅ 엑셀 원본 검증 후: "의도된 단위 분리, 문서화만 필요"
