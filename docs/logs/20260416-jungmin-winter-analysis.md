# 겨울 검증 심층 분석 — 왜 틀렸나, 어떻게 틀렸나

> 작성자: 정민 | 2026-04-16
> 대상 스크립트: `services/api/analytics/winter_analysis.py`
> 입력 데이터: 합성 2024 + 실 2025~2026 결합 학습 → 실 2025-10~2026-04 27주 검증

---

## 1. 왜 이 분석이 필요한가

지금까지 우리가 아는 건 **겨울 MAE 2,163** 이라는 **숫자 한 개**.

```
"겨울 평균 오차 2,163개"  ← 너무 거칠다
```

이걸로는 **어느 SKU에서, 어느 주에, 어떤 조건에서** 틀렸는지 모른다. 개선 방향을 잡을 수 없다.

심층 분석으로 **오차를 쪼개서** 본다:

- 주차별 오차 추이
- SKU별 오차
- 과대/과소 편향
- 날씨 조건별 오차
- 월별 오차

## 3. 핵심 발견 — 월마다 정반대의 편향

```
10~12월 (시즌 시작+피크): 전부 과소 예측
  10월: -48.4% 과소
  11월: -25.1% 과소
  12월: -33.7% 과소  ← 실제 652K vs 예측 432K (220K 차이)

1~4월 (시즌 종료+비시즌): 전부 과대 예측
   1월:  +7.4% 과대
   2월:  +162.2% 과대  ← 실제 84K vs 예측 219K
   3월:  +58.5% 과대
   4월:  +78.9% 과대
```

### 해석

모델이 **극단값을 회피하고 중간으로 수렴**하려는 경향. 이건 LightGBM 같은 tree 모델의 **일반적인 한계** — 학습 데이터의 평균 근처로 예측을 당기는 성질(regression toward the mean).

**비유**: 시험 문제 중 쉬운 건 다 풀지만 극단적으로 어려운 문제(12월 피크)와 엄청 쉬운 문제(2월 비시즌)를 양쪽으로 틀리는 학생.

## 4. 전체 편향 — 거의 균형

```
27주 중:
  과대 예측: 14주 (51.9%)
  과소 예측: 13주 (48.1%)
```

전체적으로는 편향 없음. **하지만 시점별로는 심각한 편향**이 숨어있다 (위 3번).

## 5. 날씨 조건별 MAE

```
한파 있음 (57 SKU-주): MAE 2,634
한파 없음 (나머지):   MAE 1,580
  → 한파 주가 67% 어려움

눈 있음 (202 SKU-주): MAE 2,801
눈 없음 (나머지):    MAE   976
  → 눈 온 주가 187% 어려움
```

### 해석

**극단 날씨일수록 예측 어려워짐**. 이유:

- 한파·강설 데이터가 학습에서 적음 (희소 사건)
- 합성 데이터로 채웠지만, 합성 겨울이 1.6배 과대 → 모델이 혼란
- 실제 한파 주의 판매 변동이 너무 큼 (분산이 큼)

## 6. SKU별 MAE — "평균 MAE 2,163"의 진실

### 오차 큰 SKU Top 5 (모두 상위 판매 SKU)

| SKU      | MAE        | 주 평균 판매 | 오차율 |
| -------- | ---------- | ------------ | ------ |
| 63575566 | **15,597** | 26,050개     | 59.9%  |
| 63216406 | 5,787      | 15,898개     | 36.4%  |
| 63448701 | 4,726      | 6,513개      | 72.6%  |
| 2298273  | 3,078      | 5,209개      | 59.1%  |
| 63407190 | 1,085      | 1,741개      | 62.3%  |

→ **상위 SKU는 절대 오차는 크지만 %로는 36~72%**. 수천 개 단위 오차라 **재고 의사결정에 영향 큼**.

### 오차 작은 SKU Top 5 (모두 소량 SKU)

| SKU      | MAE | 주 평균 판매 | 오차율  |
| -------- | --- | ------------ | ------- |
| 40851591 | 147 | 4개          | 3,685%  |
| 49118115 | 144 | 1개          | 14,400% |
| 28476250 | 141 | 2개          | 7,050%  |
| 51745713 | 137 | 2개          | 6,850%  |
| 50117045 | 117 | 1개          | 11,700% |

→ **소량 SKU는 MAE 140 수준으로 작아 보이지만, % 기준으로는 쓸모없는 수준** (판매 자체가 1~4개인데 오차가 100 이상).

### 교훈

**"평균 MAE 2,163"은 상위 SKU에 편향**된 숫자. 상위 5개 SKU가 전체 판매의 대부분을 차지해서 MAE 평균을 끌어올린다.

**실제 의사결정엔**:

- 상위 SKU: 36~72% 오차 → **개선 여지 큼**
- 소량 SKU: 예측 자체가 무의미 → **평균으로 통계적으로만 채우는 게 나음**

## 7. 오차 큰 주차 Top 5 (winter_analysis_weekly.csv 중)

분석 결과에서 가장 틀린 주차들의 특징:

- 시즌 전환점(10월 말, 12월 초, 2월 초) 집중
- 기온 급강하 직후 또는 첫 한파 주차

→ 모델이 **계절 전환 시점을 못 잡음**. 첫 한파 신호를 학습 피처에 넣어야 할 필요성 시사 (향후 `first_snow_flag` 추가 후보).

## 8. 합성 데이터 과대 보정을 하지 않는 이유

기존 계획: "합성 12월이 실데이터보다 1.6배 높으니 0.63배로 축소"

**분석 결과 이 계획은 잘못**:

- 모델은 **겨울 피크를 과대가 아니라 과소 예측** 중
- 12월 실제 652K → 모델 예측 432K (-33.7%)
- 합성을 더 줄이면 모델이 겨울을 **더 과소 예측**할 가능성

→ **합성 겨울 보정 보류**. 근본 문제는 "1년치 데이터로는 겨울 전환점 학습 불가"이지 "합성이 과하다"가 아님.

## 9. 개선 방향 (향후 작업)

| 우선 | 항목                        | 이유                                           |
| ---- | --------------------------- | ---------------------------------------------- |
| 1    | `first_snow_flag` 피처 추가 | 10월/11월 시즌 시작을 모델이 명시적으로 학습   |
| 2    | 분위 회귀(quantile)         | SKU별 맞춤 신뢰구간으로 "소량 SKU는 넓게" 처리 |
| 3    | 월별 SKU 분배 비율 변동     | 시즌 초/말에 인기 SKU가 다름 반영              |
| 4    | 2026년 겨울 실데이터 축적   | 근본 해결                                      |

**발표 시점에서**: 한계는 명시하되, "처음으로 겨울 MAE 측정 가능해짐"이 성과. 다음 시즌 데이터 쌓이면 재검증 가능.

## 10. 방법론 요약 (계산 근거)

### 학습 데이터

- **합성 2,108행**: `synthetic_2024_weekly.csv` (2024-01~2025-03, 66주 × 34 SKU, 실측 필터링 후)
- **실 342행**: `weekly_feature_table.csv` 중 2025-04~09 (쿠팡 일간종합성과지표 기반)

### 검증 데이터

- **실 511행**: 2025-10 ~ 2026-04 (쿠팡 일간종합성과지표 **실측만**)
- 이 중 **겨울(11~1월)** = 295행

### 모델

- LightGBM (원작자 복원 코드 `weekly_demand_forecast.py`)
- 피처 15개 (lag 3개 + 날씨 7개 + 바이박스 3개 + 프로모션 1개 + 일교차 1개)
- val 기준 early stopping, num_boost_round 500

### MAE 계산

```
MAE(주, SKU) = |실측 - 예측|
평균 MAE = Σ|실측 - 예측| / 관측수

값별 해석:
  상위 SKU (주 판매 5,000개+): MAE 5,800 ≈ 판매의 36% 오차
  중위 SKU (주 판매 100~500): MAE 500 ≈ 판매의 50~100% 오차
  소량 SKU (주 판매 1~10): MAE 140 ≈ 판매의 1000%+ 오차 (의미 없음)
```
