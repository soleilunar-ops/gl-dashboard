### 3.4 검증 결과

| 라운드 | 변경 사항                 | train MAE | val MAE        |
| ------ | ------------------------- | --------- | -------------- |
| R1     | 기본 (15피처, 마스킹 off) | 1,109     | 638            |
| R2     | + 품절 마스킹 on          | 1,115     | 645            |
| R3     | + 바이박스 가격 피처      | 1,277     | 678            |
| **R4** | **+ wind_mean**           | **1,287** | **636** ← 최저 |
| R5     | + review/rating (제거함)  | 2,102     | 1,615 ← 악화   |

**val MAE 636 = 비시즌(2~4월) 정확도**. 겨울 정확도는 데이터 부족으로 미측정.

### 3.5 신뢰구간

| 항목       | 값                     | 근거                                         |
| ---------- | ---------------------- | -------------------------------------------- |
| 계수 1.645 | 정규분포 90% z값       | 통계학 공식 [B]                              |
| MAE 기반   | val_mae × 1.645        | 간이 추정 (MAE ≠ σ, 약 20% 넓은 보수적 구간) |
| SKU별 차이 | 미반영 (전체 MAE 사용) | **한계**: 대량 SKU는 정확, 소량 SKU는 부정확 |

### 3.6 R5 실험에서 review/rating 제거한 이유

review_count는 판매의 **결과**(많이 팔리면 리뷰 늘어남)이지 **원인**이 아님. 피처로 넣으면 "리뷰 많으면 잘 팔린다"는 허위 패턴 학습 → val MAE 636→1,615 급등. **동시지표(concurrent indicator) 편향**.

## 4. Model B — 값·변수·로직 상세

### 4.1 구조

```
Model A 예측 (SKU별 주간 판매) → 카테고리 합산 → × 발주/판매 비율 → 카테고리 발주
  → SKU 분배 (직전 4주 판매 비율)
```

### 4.2 사용 값

| 값                               | 출처                     | 유형                              |
| -------------------------------- | ------------------------ | --------------------------------- |
| 발주/판매 월별 비율 (0.16~62.95) | 납품률 ÷ 일간성과 매칭   | **실데이터** [D]                  |
| SKU 분배 비율                    | 직전 4주 SKU별 판매 비중 | **실데이터** [D]                  |
| ratio_lookback_weeks = 8         | 개발자 설정              | **근거 불충분** (왜 8인지 미검증) |
| sku_distribute_weeks = 4         | 개발자 설정              | **근거 불충분** (왜 4인지 미검증) |

### 4.3 선형 회귀 vs 비율 모델

- 선형 회귀: MAE 41,503 — 48행 과적합, **사용 비권장**
- 비율 모델: 안정적, **사용 권장**
