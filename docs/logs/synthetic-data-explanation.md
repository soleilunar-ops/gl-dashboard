# 합성(더미) 데이터 생성 — 논리 구조와 계산식

> 이 문서는 중학생도 이해할 수 있도록 작성되었습니다.

## 왜 더미데이터를 만들었나?

핫팩은 **겨울에 팔리는 상품**입니다. 예측 모델이 "겨울에 얼마나 팔릴지" 잘 맞추는지 확인하려면 **겨울 시험 데이터**가 필요합니다.

그런데 우리가 가진 쿠팡 판매 데이터는 **2025년 4월~2026년 4월 (1년)**뿐입니다.

```
2025년 봄 ──── 여름 ──── 가을 ──── ★겨울★ ──── 2026년 봄
  ↑ 데이터 시작                                    ↑ 데이터 끝
```

이 1년을 "공부용(학습)"과 "시험용(검증)"으로 나누면:

- 공부에 겨울을 넣으면 → 시험에 겨울이 없음
- 시험에 겨울을 넣으면 → 공부에 겨울이 없음

**어떻게 나눠도 겨울 시험이 불가능합니다.**

해결법: **2024년 판매 데이터를 만들자!** 2024년 날씨는 실제 기상청 데이터가 있으니, 그 날씨에 맞춰 "이 정도 날씨면 이 정도 팔렸을 것"이라고 추정하는 겁니다.

## 무엇을 사용했나?

| 데이터            | 진짜/가짜       | 출처                                              |
| ----------------- | --------------- | ------------------------------------------------- |
| 2024년 **날씨**   | **진짜**        | 기상청 ASOS 5개 관측소 (서울/수원/부산/대전/광주) |
| 2025년 **판매량** | **진짜**        | 쿠팡 일간종합성과지표                             |
| 2024년 **판매량** | **가짜 (합성)** | 위 두 개를 조합해서 생성                          |
| **지역별 가중치** | **진짜**        | 쿠팡 지역별판매트렌드 (수도권 61.5%)              |
| **납품 비율**     | **진짜**        | 쿠팡 납품률 데이터 (월별 발주/판매 비율)          |
| **첫 한파 날짜**  | **진짜**        | 2024년 11월 27일 (기상청 발표)                    |

## 어떤 순서로 만들었나?

```
[Step 1] 2025년 실제 판매에서 "기온 → 판매" 관계식 추출
    ↓
[Step 2] 2024년 실제 날씨를 지역 가중 평균으로 정리
    ↓
[Step 3] 관계식 + 날씨 + 도메인 규칙 → 2024년 합성 판매량
    ↓
[Step 4] 합성 판매 + 월별 비율 → 합성 납품 데이터
```

---

## Step 1. "기온이 떨어지면 핫팩이 얼마나 더 팔릴까?" 관계식 추출

2025년 실데이터에서 컴퓨터가 찾은 관계:

```
기온 1℃ 떨어지면 → 주간 판매 약 3,279개 증가
풍속 1m/s 증가하면 → 약 20,226개 증가 (체감온도 효과)
일교차 1℃ 커지면 → 약 1,378개 증가
눈 1cm 쌓이면 → 약 2,028개 증가
```

**비유**: 수학 시험에서 "공부 시간이 1시간 늘면 점수가 5점 오른다"는 관계식을 찾는 것과 같습니다. 여기서는 "기온이 1도 떨어지면 판매가 3,279개 늘어난다"입니다.

---

## Step 2. 날씨를 "판매 비중"에 맞게 가중 평균

핫팩의 **61.5%가 수도권**에서 팔립니다. 부산이 따뜻해도 서울이 추우면 전체 판매는 늘어야 합니다.

```
단순 평균 (이전 방식):
  (서울 -3℃ + 부산 5℃ + 대전 0℃ + 광주 2℃ + 수원 -2℃) ÷ 5 = 0.4℃

판매 가중 평균 (개선된 방식):
  서울 -3℃ × 23.7%
+ 수원 -2℃ × 30.2%  (경기도 대표)
+ 부산  5℃ ×  6.5%
+ 대전  0℃ ×  2.7%
+ 광주  2℃ ×  2.7%
= -1.4℃
```

**왜 다른가**: 매출의 61.5%가 수도권이니까, 수도권 날씨가 더 중요합니다. 단순 평균(0.4℃)보다 가중 평균(-1.4℃)이 더 추운 쪽으로 나오는데, 이게 실제 핫팩 수요를 더 잘 반영합니다.

---

## Step 3. 합성 판매량 계산

각 주마다 이런 식으로 계산합니다:

### 3-1. 기본 판매량 = 12월 평균 × 월별 계절 계수

```
12월 실제 주 평균: 130,342개 (기준값)

월별 계수:
  12월: 1.00 (피크 = 기준)
  11월: 0.60
   1월: 0.55
  10월: 0.15
   2월: 0.10 ← 춥더라도 "곧 봄이니까" 수요 둔화
   3월: 0.03
   4~9월: 0.01 이하 (거의 안 팔림)

예: 11월 첫째 주
  기본 판매량 = 130,342 × 0.60 = 78,205개
```

**비유**: 아이스크림 가게에서 "8월이 100이면 12월은 5 정도"라는 계절 비율을 정하는 것.

### 3-2. 기온 효과 = "12월 평균(0℃)보다 얼마나 더 추운가"

```
12월 평균기온 ≈ 0℃ (기준점)

계산: 기온효과 = 1.0 + (0℃ - 이번주 기온) × 0.02

예: 이번 주 기온 -5℃
  기온효과 = 1.0 + (0 - (-5)) × 0.02 = 1.10 (10% 증가)

예: 이번 주 기온 15℃
  기온효과 = 1.0 + (0 - 15) × 0.02 = 0.70 (30% 감소)
```

### 3-3. 풍속 효과 = "바람이 세면 더 춥게 느껴짐"

```
풍속효과 = 1.0 + max(0, 풍속 - 2) × 0.02

풍속 2m/s 이하: 효과 없음
풍속 5m/s: 1.0 + 3 × 0.02 = 1.06 (6% 증가)
```

### 3-4. 일교차 효과

```
일교차효과 = 1.0 + max(0, 일교차 - 10) × 0.01

일교차 10℃ 이하: 효과 없음
일교차 15℃: 1.0 + 5 × 0.01 = 1.05 (5% 증가)
```

### 3-5. 눈 효과

```
눈효과 = 1.0 + min(적설량, 20cm) × 0.05

눈 0cm: 효과 없음
눈 10cm: 1.0 + 10 × 0.05 = 1.50 (50% 증가!)
```

### 3-6. 첫 한파 충격 (2024년 11월 27일)

```
첫 한파 주: 2.5배 급등 (갑자기 "핫팩 사야지!" 심리)
다음 주: 1.5배 (여파)
2주 뒤: 1.2배 (약한 여파)
그 이후: 1.0배 (정상)
```

**비유**: 매년 첫 눈이 오면 "아 겨울이다!" 하고 핫팩을 사는 것. 두 번째 눈부터는 "이미 샀으니까" 덜 삼.

### 3-7. 최종 합산

```
이번 주 판매량 = 기본판매 × 기온효과 × 풍속효과 × 일교차효과 × 눈효과 × 첫한파충격

예: 2024년 11월 27일 주차 (첫 한파)
  기본: 130,342 × 0.60 = 78,205
  기온(-3℃): × 1.06
  풍속(4m/s): × 1.04
  일교차(12℃): × 1.02
  눈(5cm): × 1.25
  첫한파: × 2.50

  = 78,205 × 1.06 × 1.04 × 1.02 × 1.25 × 2.50
  = 약 270,000개
```

### 3-8. SKU 분배

전체 판매량을 개별 제품에 나눕니다.

```
전체: 270,000개

실데이터 기준 SKU 비중:
  하루온 붙이는 핫팩: 38.1% → 102,870개
  박상병 핫팩:       12.5% → 33,750개
  불가마 핫팩:        8.7% → 23,490개
  ...

= 각 SKU별 1행씩 생성
```

### 3-9. 랜덤 노이즈 추가

실제 판매는 매주 조금씩 다릅니다. "딱 맞는 숫자"만 생성하면 너무 깨끗해서 비현실적.

```
판매량 = 계산값 + 랜덤변동 (±실데이터 잔차의 30% 수준)

예: 계산값 270,000 ± 약 9,200 → 260,800 ~ 279,200 사이
```

## Step 4. 합성 납품 데이터

판매 예측에서 "쿠팡이 얼마나 발주할지"를 추정합니다.

### 월별 발주/판매 비율 (실데이터 기준)

```
 9월: 판매의 63배 발주 ← 시즌 대비 선발주 (겨울 준비)
10월: 3.5배
11월: 2.4배
12월: 1.5배 (피크, 판매 따라잡기)
 1월: 0.4배 (시즌 하강)
 2월: 0.16배 (거의 발주 안 함)
 4~7월: 0 (비시즌, 납품 없음)
 8월 말: 납품 재시작 (다음 겨울 준비)
```

```
예: 12월 합성 판매 = 1,037,537개
  → 합성 발주 = 1,037,537 × 1.54 = 1,597,806개
```

## 한계 (정직한 평가)

| 항목                   | 설명                                                                                                       |
| ---------------------- | ---------------------------------------------------------------------------------------------------------- |
| **12월 과대추정**      | 합성 12월 주 평균 182,300개 vs 실데이터 130,342개 (약 1.4배). 기온/풍속 효과가 계절 계수와 일부 이중 반영. |
| **알 수 없는 이벤트**  | 2024년에 있었을 수 있는 프로모션, 경쟁사 행동, 쿠팡 정책 변경 등은 반영 불가.                              |
| **SKU 비율 고정**      | 모든 주에 동일 SKU 비율 적용. 실제론 시즌 초반/말 인기 제품이 다를 수 있음.                                |

| **합성으로 검증 금지** | 합성 데이터로 만든 시험지로 시험보면 의미 없음. 학습 보강 용도로만 사용.                                   |

## 이 데이터로 뭘 할 수 있나?

```
합성 2024년 (학습 보강) + 실데이터 2025년 = 2년치
  → 학습: 2024년 겨울 (합성) + 2025년 봄~가을 (실제)
  → 검증: 2025년 겨울 (실제)
  → 비로소 "겨울 예측 정확도" 측정 가능!
```
