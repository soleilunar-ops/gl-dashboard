# 정민에게 — Supabase 연동 가이드 v2

작성자: 김지호 (조장)
작성일: 2026-04-17 (v2 수정)
대상: 정민 (수요 예측 모델 담당)

## v1과 다른 점

v1을 AI 지시서로 만들어서 Supabase에 실제로 DDL을 실행해보면서 검증했어. 그 과정에서 **내가 v1에서 잘못 안내한 부분 2가지 + 경미한 문제 3가지**가 있어서 v2로 고쳤어. 특히 날씨 데이터 부분이 핵심.

---

## 1. 핵심 결론 3줄

1. **Supabase에 날씨 데이터는 2종류(ASOS + ERA5) 둘 다 필요하다** — ASOS는 풍속만, ERA5는 강수/눈만 데이터가 있음. 둘 중 하나만 쓰면 겨울 검증이 터짐
2. **이미 Supabase에 있는 테이블 5개는 그대로 쓸 수 있다** — 정민 로컬 CSV 중 5개는 쿼리로 대체 가능
3. **모델 산출물 4가지는 Supabase에 없다** — 신규 테이블 4개를 새로 만들어야 함

---

## 2. v1에서 수정한 내용

### 🔴 치명적 수정 1: 날씨 데이터는 **하이브리드**가 정답

v1에서 내가 "ASOS 쓰거나 ERA5 쓰거나 선택"이라고 했는데, 실제로 Supabase에 넣어놓고 검증해보니 **둘 다 필요**해:

| 필요한 피처                 | ASOS      | ERA5             |
| --------------------------- | --------- | ---------------- |
| 평균기온(temp_avg)          | ✅ 100%   | ✅ 100%          |
| 평균 풍속(wind_avg)         | ✅ 99%    | ❌ **0% (NULL)** |
| 강수량(rain, precipitation) | ❌ **0%** | ✅ 100%          |
| 적설량(snowfall)            | 부분      | ✅ 100%          |

즉 **ASOS에서 온도/풍속, ERA5에서 강수/눈**을 가져와서 조인해야 모든 피처가 채워져. v2에는 이 하이브리드 패턴 실코드를 넣었어 (AI용 문서 TASK_2.1 참조).

실측 검증 결과: 2025-11 ~ 2026-03 기간 755행 중 temp 755/755, wind 750/755 (99.3%), rain 755/755, snow 755/755 완벽 매칭.

### 🔴 치명적 수정 2: `v_sales_weather` 뷰 쓰지 말 것

v1에서 "Model A 학습 입력 한 번에 해결"이라고 추천했는데 이 뷰 실제 정의를 보니 **서울 관측소 + ASOS만** 조인해. 정민 모델은 5개 관측소 쓰고 rain도 필요한데 이 뷰 쓰면 둘 다 없음. 직접 조인 코드로 교체했어.

필요하면 내가 `v_weather_hybrid` 뷰를 만들어줄 수 있어. 이쪽이 더 편하면 알려줘.

### 🟡 경미한 수정 3가지

- **PostgREST alias**: `.select("date AS op_date")` 같은 문법은 Supabase Python에서 안 먹어. pandas rename으로 처리하도록 수정
- **Delivery 컬럼 rename**: v1은 주석만 있었는데 실제 rename 코드 추가
- **upsert 호환**: `supabase-py` 버전에 따라 `on_conflict` 파라미터 시그니처 달라서 try/except 폴백 추가

## 3. 정민이 선택해야 할 것 (의사결정 5가지)

### ① 날씨 데이터 하이브리드 방식에 동의?

기본 권고: ASOS(온도·풍속) + ERA5(강수·눈) 하이브리드.
동의하면 내가 `v_weather_hybrid` 뷰를 만들어서 더 간단하게 쓸 수 있게 할게.

### ② 바이박스 데이터 업로드 주기?

- A: 1회성 (과거 5개월치만)
- B: 매일 신규 수집
- C: 주 단위 배치
  선호하는 거 알려줘. 스크래핑 자동화까지 이어지면 좋겠지만 복잡도 있음.

### ③ model_version 컬럼 유지?

추천: 유지. A/B 실험할 때 편하고 저장공간 부담 없음.

### ④ 합성 2024 데이터 Supabase 업로드?

추천: **로컬 유지**. 학습 전용이고 대시보드가 안 쓰니까 Supabase 공간 낭비야. Git LFS로 관리 권장.

### ⑤ 지역 가중 평균 방식?

3가지 옵션 코드로 구현해놨어 (AI 문서 TASK_2.1):

- A: simple_mean (5개소 단순 평균)
- B: seoul_dominant (서울 61.5% 가중, 정민 기존 synthetic 로직과 동일) ← **추천**
- C: capital_regional_60 (수도권 60% 가중)
