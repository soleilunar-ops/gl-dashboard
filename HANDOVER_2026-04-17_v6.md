## 🔑 중요 SQL 스니펫

### 매핑 resolve (DISTINCT ON 최신귀속) — orders JOIN 때 핵심

```sql
SELECT DISTINCT ON (erp_system, erp_code)
  erp_system, erp_code, item_id
FROM item_erp_mapping
WHERE erp_code IS NOT NULL
ORDER BY erp_system, erp_code,
  CASE confidence WHEN 'HIGH' THEN 1 WHEN 'MEDIUM' THEN 2
                  WHEN 'LOW' THEN 3 WHEN 'CHECK' THEN 4 ELSE 5 END,
  item_id DESC  -- 최신 제조년도 = 큰 item_id
```

### 법인별 × 거래유형별 집계

```sql
SELECT erp_system, tx_type, COUNT(*), SUM(quantity), SUM(total_amount)
FROM orders GROUP BY 1,2 ORDER BY 1,2;
```

### 판매 기록 없는 144품목 확인

```sql
SELECT im.seq_no, im.item_name_raw, im.base_stock_qty
FROM item_master im
LEFT JOIN orders o ON o.item_id = im.item_id
WHERE o.id IS NULL
ORDER BY im.seq_no;
```

### 현재 재고 조회

```sql
SELECT * FROM v_current_stock ORDER BY seq_no;
```

### 쿠팡 매출 + 재고 통합

```sql
SELECT * FROM v_item_with_coupang_status
WHERE has_coupang_mapping = TRUE
ORDER BY gmv_30d DESC NULLS LAST;
```
